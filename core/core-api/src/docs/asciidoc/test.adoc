== 에러 처리

=== 공통 에러

다음 표는 API에서 반환하는 공통 에러 코드와 메시지를 나열합니다.

|===
| 코드 | 메시지 | 설명
| 400  | 잘못된 요청 | 요청을 이해할 수 없거나 필수 매개변수가 누락되었습니다.
| 401  | 인증 실패 | 인증에 실패했거나 사용자가 원하는 작업에 대한 권한이 없습니다.
| 404  | 찾을 수 없음 | 요청한 리소스를 찾을 수 없습니다.
| 500  | 서버 내부 오류 | 서버에 오류가 발생했습니다.
|===

=== 에러 응답 예시

[source,json]
----
{
  "error": "잘못된 요청",
  "message": "요청을 이해할 수 없거나 필수 매개변수가 누락되었습니다.",
  "status": 400
}
----

== 보안 고려사항

이 섹션에서는 API 사용 시 보안 조치 및 모범 사례를 설명합니다.

=== 인증

모든 API 요청에는 `Authorization` 헤더에 유효한 JWT 토큰이 포함되어야 합니다.

[source,http]
----
Authorization: Bearer {token}
----

=== 데이터 보호

- 클라이언트와 서버 간 전송되는 데이터를 암호화하기 위해 항상 HTTPS를 사용합니다.
- 강력한 해싱 알고리즘과 솔트를 사용하여 비밀번호를 안전하게 저장합니다.
- SQL 인젝션 공격을 방지하기 위해 입력값 검증 및 정화를 구현합니다.
